Grocery Bud ToDo List

A javascript CRUD application using local storage.

Grocery bud

Project Description

Using vanilla JS, I created this ToDo app, which essentially covers all the basic fucntions of JavaScript, to create, read, update and delete "ToDo's" to and from a locally stored object.

I decided against using a framework (really into Svelte atm) because I wanted to prove to myself that I was capable of using JS alone, without all the fancy plugins / frameworks to do all the lifting for me.

There are some additional CSS features which act as alerts to the user notifing them when an item has been added, removed from their list, to aid in user understanding of the app's current state.

Challenges

One challenge involved adding editing to a selected ToDo, where it would be modified on the screen but also locally, deleting the local storage ID of the previous value.

Another challenge involved addign the local storage itself, using JSON.Stringify() to save the content as strings in order to edit them effectively.

Dynamically adding in the html at runtime was another hurdle as I had to understand Templating in order to perform this well.

Solutions

To dynamically add the html in I used templated strings, which proved to be very useful.

I created a setBackToDefault() function that I called at the end of every edit / delete / add ToDo, to restore the system state to the start. This made the development a lot simpler as I didn't need to repeatedly set all these values.

After understanding DOM navigation, editing the current event target's sibling elements proved to be a much easier task. Allowing me to access the Todo 'grandparent' container to get the dataset.id and remove a CSS property that was showing the container. Then removing it within the JS.

I decided to make the webpage be changed first through CSS hiding elements, then remove using the JS, in case the user was on a slow network.
